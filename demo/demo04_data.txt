:= What is mruby?
:
  * 組み込み向きの小さなRuby実装

:
  * 小さな環境でも動く

:= Ruby VM
:
  * Virtual Machine
    a machine inside a machine
    "仮想"マシン
:
  * Ruby source is compiled
    into bytecode first
:
  * The VM runs the bytecode

:= What is bytecode?
:
  * VM instructions
:
  * シンプルなインストラクション
:
  * mrubyはレジスタまわりのインストラクションが多い

:= Bytecodeの例

  Ruby:
:
    return 3 == 2
:
  Bytecode:
:
  1 000 LOADI_3    R1
  1 002 LOADI_2    R2
  1 004 EQ         R1    R2
  1 006 RETURN     R1
  1 008 STOP

:= Bytecode usage
:  mrbcはC用ヘッダーのアウトプット可
:  const uint8_t eq_int[] = {

    0x52,0x49,0x54,0x45,0x30,

    0x33,0x30,0x30,0x00,0x00,

...
:  Cで実装したVMから読み込んで実行

:= Limitations and challenges
:
  * RAMが少ない
    アロケーションを少なく!
:
  * 32-bitサポートがない
:
  * VMとアプリケーションの問題の切り分け
:
  * Slow, but not unusable

:= タイマー実装
:
  * 時計がない
:
  * z88dkはvsyncのカウントをしてくれている
    1秒につき60
:  * 16ビットしかないので65535まで
:    つまり約1092秒で0秒に
:
  * floatがない
    * 19 / 25 * 31
      floatを使うと23.56,整数なら0
:    * 19 * 100 / 25 * 31 / 100
     これなら整数で23
: * vmがuint16_tを扱えない

:= 日本語サポート
:
  * すべての文字はタイルである
:
  * 448 tiles -> 9 bit!
    8bitでエンコードすると256文字
:
  * 使う文字だけROMに入れたい
:
  * カスタムエンコーディング
:
  * Cからのエラーメッセージは
    コンパイルし直さないと読めない
:
  * 変換ツールをCoding Agentに書いてもらった
:
  * 文字種が256種を超えた
