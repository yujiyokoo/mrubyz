:---  What is mruby?
:
  * smaller, embeddable
    Ruby implementation

:
  * integrates well with C and
    other languages
:
  * suitable for small
    environments

:---  What is Ruby VM
:
  * Virtual Machine
    a machine inside a machine
:
  * When Ruby is run, the
    source is compiled into
    VM instructions first
:
  * The VM runs the bytecode

:---  What is bytecode?
:
  * VM instructions
:
  * Simple instructions
:
  * Register-level ops
    for mruby

:---  Bytecode example

  Ruby:
:
    return 3 == 2
:
  Bytecode:
:
  1 000 LOADI_3    R1
  1 002 LOADI_2    R2
  1 004 EQ         R1    R2
  1 006 RETURN     R1
  1 008 STOP

:---  Bytecode usage
:  mrbc can produce bytecode
  in C-compatible integer array
:  const uint8_t eq_int[] = {

    0x52,0x49,0x54,0x45,0x30,

    0x33,0x30,0x30,0x00,0x00,

    0x00,0x45,0x4d,0x41,0x54,
...
:  include in your C source
:  read from your vm

    mrubyz

:---  What is Z80?
:
  * Popular processor from the
    80s
:
  * Primarily 8-bit
:
  * Used for many platforms

    * Sega Master System

    * Sega Game Gear

    * Sinclair ZX Spectrum

    * Many more

:---  What is Master System?
:
  * Sega's Z80 console
:
  * Z80 at 3.5MHz
:
  * 8KB RAM
    * with 64KB space
    * 32KB ROM without bank
      switching
:
  * 16KB VRAM, 256x192px
    Tiles, sprites, and palettes
:
  * Mega Drive can run
    Master System games

:---  What is a minimal VM?
:
  A program that reads and

  executes bytecode
:
  (a small subset of it)
:
  No performance consideration

:---  What is supported in this VM?

:  * Built-in functions
:  * Assignment
:  * strings
:  * Add, sub, multi, div (int)
:  * Branching with if-else
:  * while loop
:  * equality comparison
:  * greater than (>)
:  * arrays
:  * Many more!

:---  What is not supported?
:  * most things
:  * less than (<), etc.
:  * garbage collection
:  * string manipulation
    (data stays in ROM)
:  * exceptions
:unsupported instruction: 0x12
OP: OP_MODULE     = 0x5D

:---  What is not supported?

  * most things

  * less than (<), etc.

  * garbage collection

  * string manipulation
    (data stays in ROM)

  * exceptions
:  * classes
:  * procs
:  * function definitions
    (coming soon!)
:  * lots more!
