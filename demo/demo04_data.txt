:= What is mruby?
:
  * Lightweight Ruby
:
  * Great for embedded and
      small environments
:
  * Many variations:

    * mruby

    * mruby/c

    * PicoRuby, FemtoRuby

:= Ruby VM
:
  * Virtual Machine
    a machine inside a machine
:
  * Ruby source is compiled
    into bytecode first
:
  * The VM runs the bytecode

:= What is bytecode?
:
  * VM instructions
:
  * Simple instructions
:
  * mruby VM is a register
    machine, so lots of
    register-based operations

:= Bytecode example

  Ruby:
:
    return 3 == 2
:
  Bytecode:
:
  1 000 {hl}LOADI_3    R1{hl}
  1 002 {hl}LOADI_2    R2{hl}
  1 004 {hl}EQ         R1    R2{hl}
  1 006 {hl}RETURN     R1{hl}
  1 008 {hl}STOP{hl}

:= Bytecode usage
:  mrbc can output C headers
:  const uint8_t eq_int[] = {

    0x52,0x49,0x54,0x45,0x30,

    0x33,0x30,0x30,0x00,0x00,

    ...
:
    Read this bytecode in the
    VM and execute

:
  * 8-bit CPU

  * 1976-2024

    * Sega Game Gear

    * MSX

:= Master System?
:
  * Sega Z80 Console, very
    similar to Mark III
:
  * Z80 at 3.5Mhz
:
  * 8KB RAM
    * 64KB Address space
    * 32KB ROM (more available)
:
  * 16KB VRAM, 256x192px
    Tiles, sprites, palettes

  * Mega DriveはMaster System互換

:
  * No gcc or clang for Z80

  (or mruby) in so many places, and
  not knowing what breaks, or if it
  would even run,

:= Limitations and challenges
:
  * Limited memory space
    Avoid allocation
:
  * No (native) 32-bit support
:
  * Is it the VM bug, or app bug?
:
  * Slow, but not unusable

:= Timer implementation
:
  * No clock available
:
  * z88dk counts vsync interrupts
    60 times per second
:  * 16 bit means max 65535
:    This means overflow to 0 at
    approx 1092s
:
  * No floating point type
    * 19 / 25 * 31
      23.56 with float, 0 with int
:    * 19 * 100 / 25 * 31 / 100
      23 with int
: * The VM can't handle uint16_t

:= Japanese support
:
  * Every character is a tile
:
  * 448 tiles -> 9 bit!
    8bit would be 256 chars
    minus special chars
:
  * Only want the used characters
    in ROM
:
  * Break the ecnoding

:
  * Got Coding Agent to write converter
:
  * This presentation was okay but
    previously it has gone over 256

:= Summary
:
  * Small subset of Ruby is still useful
:
    "useful"?
:
  * To keep it small,
      think hard about what {hl}not{hl} to support
:
  * Writing your own VM is not that hard
:
    assuming you're okay with limitations

:      return 1 + 2
