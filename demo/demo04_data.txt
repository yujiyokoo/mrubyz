:= VM
:
  * Virtual Machine
    a machine inside a machine
:
  * Ruby source is compiled
    into bytecode first
:
  * The VM runs the bytecode

:= What is bytecode?
:
  * VM instructions
:
  * Simple instructions

:= Bytecode example

  Ruby:
:
    return 3 == 2
:
  Bytecode:
:
  1 000 {hl}LOADI_3    R1{hl}
  1 002 {hl}LOADI_2    R2{hl}
  1 004 {hl}EQ         R1    R2{hl}
  1 006 {hl}RETURN     R1{hl}
  1 008 {hl}STOP{hl}

:= mrubyz features

:  * (very) incomplete

:  * Add, sub, multi, div (int)

:  * No float

:  * loop (only with 'while')

:  * equality comparison (==)

:  * greater than (>)

:  * No less than (<)
:    a <= b
:    {hl}->{hl} b > a or b == a

:  * arrays

:  * functions

:  * No GC

:  * No string manipulation
    (data stays in ROM)

:  * No exceptions
    (prints error message
     and exits)

:  * No classes / modules

:= Limitations and challenges
:
  * Limited memory space
    Avoid allocation
:
  * No (native) 32-bit support
:
  * Slow, but not unusable

:= Timer implementation
:
  * SMS has no clock
:
  * z88dk counts vsync
    interrupts 60 times per
    second

:  * 16 bit means max 65535
:    This means overflow to 0
    at approx 1092s
:
  * No floating point type

    * 19 / 25 * 31
      23.56 with float,
      0 with int

:    * 19 * 100 / 25 * 31 / 100
      23 with int
:
  * The VM can't handle
    unsigned 16 bit integer
:
  * (400 * 100) / 25 / 100
:    {hl}400 * 100 overflows!{hl}

:= {hl}# Minimal VM source{hl}
{hl}def{hl} vmrun
  pc = 48 {hl}# Skip everything!{hl}
  reg = 0 {hl}# Only one register!{hl}
  running = true

  {hl}while{hl} running
    op = read_bytecode(pc)
    pc += 1
    {hl}if{hl} op == 0x09 {hl}# LOADI_3{hl}
      a = read_bytecode(pc)
      pc += 1 {hl}# Skip{hl}
      reg = 3
    {hl}elsif{hl} op == 0x38 {hl}# RETURN{hl}
      pc += 1 {hl}# Skip{hl}
      rval = reg
    {hl}elsif{hl} op == 0x69 {hl}# STOP{hl}
      running = false
    {hl}end{hl}
  {hl}end{hl}
  {hl}return{hl} rval
{hl}end{hl}
