:= What is mruby?
:
  * Lightweight Ruby
:
  * Great for embedded and
      small environments
:
  * Many variations:

    * mruby

    * mruby/c

    * PicoRuby, FemtoRuby

:= Ruby VM
:
  * Virtual Machine
    a machine inside a machine
:
  * Ruby source is compiled
    into bytecode first
:
  * The VM runs the bytecode

:= What is bytecode?
:
  * VM instructions
:
  * Simple instructions
:
  * mruby VM is a register
    machine, so lots of
    register-based operations

:= Bytecode example

  Ruby:
:
    return 3 == 2
:
  Bytecode:
:
  1 000 {hl}LOADI_3    R1{hl}
  1 002 {hl}LOADI_2    R2{hl}
  1 004 {hl}EQ         R1    R2{hl}
  1 006 {hl}RETURN     R1{hl}
  1 008 {hl}STOP{hl}

:= Bytecode usage
:  mrbc can output C header files
:  {hl}const uint8_t{hl} eq_int{hl}[]{hl} = {

    0x52,0x49,0x54,0x45,0x30,

    0x33,0x30,0x30,0x00,0x00,

...
:  Read the bytecode in the VM and run

:= What is Z80?
:
  * 8-bit CPU
:
  * 1976-2024
:
  * Many platforms

    * Sega Master System

    * Sega Game Gear

    * MSX

    * Many more

:= Master System?
:
  * Sega Z80 Console, very similar to
    Mark III
:
  * Z80 at 3.5Mhz
:
  * 8KB RAM
    * 64KB Address space
    * 32KB ROM (more with bank switch)
:
  * 16KB VRAM, 256x192px
    Tiles, sprites, palettes
:
  * Mega DriveはMaster System互換

:= Limitations and challenges
:
  * RAMが少ない
    アロケーションを少なく!
:
  * 32-bitサポートがない
:
  * VMとアプリケーションの問題の切り分け
:
  * Slow, but not unusable

:= タイマー実装
:
  * 時計がない
:
  * z88dkはvsyncのカウントをしてくれている
    1秒につき60
:  * 16ビットしかないので65535まで
:    つまり約1092秒で0秒に
:
  * floatがない
    * 19 / 25 * 31
      floatを使うと23.56,整数なら0
:    * 19 * 100 / 25 * 31 / 100
     これなら整数で23
: * vmがuint16_tを扱えない

:= 日本語サポート
:
  * すべての文字はタイルである
:
  * 448 tiles -> 9 bit!
    8bitでエンコードすると256文字
:
  * 使う文字だけROMに入れたい
:
  * カスタムエンコーディング
:
  * Cからのエラーメッセージは
    コンパイルし直さないと読めない
:
  * 変換ツールをCoding Agentに書いてもらった
:
  * 文字種が256種を超えた
